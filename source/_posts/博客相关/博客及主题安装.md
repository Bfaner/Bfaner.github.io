Hexo是一个博客框架。Hexo使用Markdown解析文章，快速使用靓丽的主题生成静态网页。本博客所使用的版本信息如下：
Nodejs：v16.17.1
git：2.36.0.windows.1
架构：hexo 6.3.0
主题：butterfly
服务器：GitHub

# 安装
## NodeJS
Hexo基于Node.js运行。Node.js是一个JavaScript的运行环境，用于方便地搭建响应速度快、易于扩展的网络应用。使JavaScript运行在服务端的开发平台，让JavaScript成为与PHP、Python、Perl、Ruby等服务端语言平起平坐的脚本语言。
随Node.js一起安装的NPM是一个包管理工具，它具有以下功能：允许用户从NPM服务器下载别人编写的第三方包到本地使用。允许用户将自己编写的包或插件上传到NPM服务器供别人下载使用。
官网链接：https://nodejs.org/zh-cn/download/
![Nodejs下载](博客及主题安装/Nodejs下载.png)
查看版本：
```cmd
    node -v
```
为加快下载速度，将npm地址修改为淘宝镜像源
```cmd
    npm config set registry https://registry.npm.taobao.org
```
后续需要通过Node.js下载一些依赖库，因此修改Node.js默认下载安装路径。在安装目录下新建两个文件夹node_cache和node_global，并将npm下载路径修改为这两个文件夹（分别为本用户目录和全局目录）。
```cmd
    npm config set prefix "E:\Program Files\nodejs\node_global"
    npm config set cache "E:\Program Files\nodejs\node_cache"  
```
![Nodejs](博客及主题安装/Nodejs文件夹.png)
同时还需要修改环境变量，分别在Path和NODE_PATH中增加刚刚创建的两个文件夹。
![Nodejs](博客及主题安装/Nodejs环境变量.png)
## git
hexo初始化、主题下载等都要使用git。
官网链接：[Git下载](https://git-scm.com/downloads)
查看版本：
```cmd
    git --version
```
## hexo
新建一个文件夹，如 E:/BFblog，用于存放博客的相关文件。进入该文件夹，执行以下命令安装hexo。
```cmd
    npm install hexo-cli -g
```
安装成功后，在该文件夹内可能生成一些文件，直接删除。hexo的环境已经在Node.js的目录下安装完毕，此处的文件为一些依赖文件，可以在之后重新生成。
在文件夹中执行如下命令使hexo初始化，这一过程中通过git下载hexo的依赖文件以及初始界面等。
```cmd
    hexo init
```
## GitHub
如果没有服务器的话，可以将博客搭建在GitHub。
在博客的文件夹中安装GitHub部署，用于将网页部署到GitHub。
```cmd
    npm install hexo-deployer-git --save
```
登录自己的github新建一个仓库，为了网页能够直接访问，仓库的名字为：xxx.github.io，其中xxx为GitHub账户名称。
![GitHub仓库](博客及主题安装/GitHub新建仓库.png)
注册完成后，打开博客的文件夹，编辑_config.yml，将deploy下面的仓库地址设置为GitHub仓库的ssh地址。
![GitHub仓库](博客及主题安装/仓库设置.png)
此时便可以将博客部署到GitHub页面上，但在每次执行部署命令时都需要输入GitHub账号和密码，为了不重复输入，在GitHub上添加ssh公钥。在命令行中输入ssh-keygen -t rsa指定RSA算法生成密钥，然后三个回车，之后就就会生成两个文件，分别为秘钥id_rsa 和公钥id_rsa.pub。复制公钥id_rsa.pub文件里的内容,进入GitHub主页，先点击右上角，再点击settings,先点击SSH and GPG keys，再点击New SSH key添加，名字随便起，Key中将id_rsa.pub的内容全部粘贴。
可以使用以下命令验证密钥是否安装成功。
```cmd
    ssh -T git@github.com
```
## 博客生成与访问
在博客的文件夹中执行如下命令
```cmd
    hexo cl # hexo clean，清除之前的文件
    hexo g  # hexo generate，生成网页
    hexo s  # hexo server，本地访问连接
    hexo d  # hexo deploy，部署到GitHub
```
hexo s后，可以通过http://localhost:4000/ 实现本地访问
hexo d后，可以通过https://bfaner.github.io/ 访问

# 主题修改
## 安装主题
github和npm都可以下载butterfly的主题，可以选择一种方式下载：
github下载稳定版本：
```cmd
git clone -b master https://github.com/jerryc127/hexo-theme-butterfly.git themes/butterfly
```
npm下载：
```cmd
npm i hexo-theme-butterfly
```
下载后的主题存放在插件文件夹node_modules的目录下，使用很不方便。因此将hexo-theme-butterfly整个文件夹剪贴到hexo/themes下，并修改文件夹名字为butterfly。
## 应用主题
修改根目录下的_config.yml，把主题改为butterfly
```yml
theme: butterfly
```
